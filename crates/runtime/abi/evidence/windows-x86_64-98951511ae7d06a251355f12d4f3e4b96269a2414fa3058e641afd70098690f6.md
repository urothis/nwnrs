### Shutdown strategy

The exported `NWNXEntryPoint` at RVA `0x00018130` fills the engine-global
pointer table, including the four debug globals above, but explicitly writes
null to the exit-program slot at offset `0x70`.

Windows `WinMain` at RVA `0x00017080` exits its server loop through the Windows
message queue rather than the Linux `g_bExitProgram` global. Schema 2 therefore
records `administration.shutdown.strategy = "current_thread_message_queue"`.
The administration bridge services that operation synchronously on the engine
thread by posting `WM_QUIT`. No guessed data RVA is used.

## Native verification

The pinned Unified ABI probe was compiled natively with MSVC x64. Its complete
snapshot matches the source-controlled target pack, including the Windows-only
`CNWSPlayerTURD` offsets (`776`, `792`, `808`), creature-stats offset (`2768`),
and 160-byte `CVirtualMachineScript` layout.

The Windows Frida Gum probe and the MSVC native runtime fixture both pass. The
fixture covers PE identity, DLL startup, hook installation, stack transport,
non-trivial `CExoString` return ownership, server state, event context,
administration calls, and graceful shutdown.

On 2026-07-19, the source-controlled module was also launched through `nwnrs`
against the exact Steam `nwserver.exe` hash above. The runtime selected this
pack, initialized before the server resumed, reported the expected
`module.load` event, API version, build, platform, module/player
state, and UDP port, then handled a launcher `quit` request and exited cleanly
with status 0.
