### Shutdown exception

The current `administration.exit_program` contract cannot be populated safely
for this executable. The exported `NWNXEntryPoint` at RVA `0x00018130` fills
the engine-global pointer table, including the four debug globals above, but
explicitly writes null to the exit-program slot at offset `0x70`.

Windows `WinMain` at RVA `0x00017080` exits its server loop through the Windows
message queue rather than the Linux `g_bExitProgram` global. Windows shutdown
therefore needs a platform-specific operation such as posting `WM_QUIT` on the
engine thread, or a revised target-pack shutdown strategy. A guessed data RVA
must not be substituted for this field.

## Remaining verification

These RVAs establish exact identity and address selection. Before release they
still require the normal native fixture and real-server checks for the MSVC
calling convention, non-trivial `CExoString` ownership, class layouts, hook
installation, and behavior.
